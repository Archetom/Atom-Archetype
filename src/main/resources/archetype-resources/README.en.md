# ${rootArtifactId}

[简体中文](README.md)

`${rootArtifactId}` is a Java 25 and Spring Boot 4.1 multi-module Maven project generated by Atom Archetype. It includes a working user-management example.

Generated coordinates: `${groupId}:${rootArtifactId}:${version}`. Run `make clean-sample` if you do not need the example code.

$h2 Requirements

- JDK 25
- Bundled Maven Wrapper 3.9.16
- Docker with Docker Compose for local MySQL and integration tests
- Make (optional)

Redis is disabled by default and is not required to build or start the project.

$h2 Quick start

Start MySQL and build all modules:

```bash
docker compose up -d mysql
sh ./mvnw clean install
```

Run the application with the `dev` profile:

```bash
ATOM_SECURITY_TRUSTED_HEADER_ENABLED=true \
  sh ./mvnw -f start/pom.xml spring-boot:run -Dspring-boot.run.profiles=dev
```

Check the health endpoint:

```bash
curl http://localhost:8080/actuator/health
```

Configuration files are stored in `conf/`. Development API documentation is available at `http://localhost:8080/swagger-ui/index.html`.

$h2 Common commands

| Command | Purpose |
|---|---|
| `make compile` | Compile all modules |
| `make test` | Run tests that do not require Docker |
| `make integration-test` | Run the full suite with MySQL Testcontainers |
| `make infra-up` | Start local MySQL and Redis |
| `make run` | Run the application with the `dev` profile |
| `make clean-sample` | Remove the bundled User example |

If Make is not installed, see `Makefile` and run the corresponding Maven Wrapper, Docker Compose, or `clean.sh` command directly.

$h2 Project structure

| Module | Responsibility |
|---|---|
| `api` | Public requests, responses, facade contracts, and `AuthenticatedCaller` |
| `domain` | Aggregates, value objects, domain events, repository and domain-service ports |
| `application` | Use-case orchestration, command/query templates, output ports, and transaction callbacks |
| `shared` | Framework-neutral result and error types |
| `infra/rest` | Spring MVC, Spring Security, and HTTP error mapping |
| `infra/persistence` | MyBatis-Plus, Flyway, and cache adapters |
| `infra/external` | Third-party system adapters |
| `infra/security` | Password hashing and other security adapters |
| `infra/facade` | Facade contract implementations |
| `start` | Spring Boot composition root and integration tests |

Dependencies flow from `infra` to `application` to `domain`. The `domain` module does not depend on `application`, `api`, or any `infra` module. See [Architecture](docs/architecture.md) for the complete rules.

$h2 Configuration

- `conf/application-dev.yml`: local development settings.
- `conf/application-test.yml`: automated test settings.
- `conf/application-prod.yml`: production settings, with no default datasource credentials.

The `X-Dev-User-Id` and `X-Dev-Tenant-Id` headers are limited to explicitly enabled `dev` and `test` environments. Production should integrate a real identity provider and map verified identities to `AuthenticatedCaller`.

Enable Redis with `atom.redis.enabled`; the default is a no-op cache implementation. Flyway migrations live in `infra/persistence/src/main/resources/db/migration`; add a new migration for database changes instead of editing an applied migration.

See [Configuration](docs/configuration.md) for identity, Redis, database, and secret settings.

$h2 Tests

```bash
sh ./mvnw test
CI=true sh ./mvnw test   # requires Docker
```

Integration tests cover Flyway, tenant isolation, optimistic locking, soft deletion, HTTP authentication, and error mapping. The current test target is JVM deployment; GraalVM native-image is not included.

$h2 Remove the example

The bundled User aggregate demonstrates the project layers and test setup. Remove it when it is not needed:

```bash
make clean-sample
```

The command uses `bash clean.sh` to remove the example files while preserving each module directory. Run `sh ./mvnw test` again afterwards.
